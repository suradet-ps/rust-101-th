## Part 00: Algebraic Datatypes (ชนิดข้อมูลแบบพีชคณิต)

สำหรับโค้ด Rust ชิ้นแรกของเรา เราจะมาเขียนฟังก์ชันคำนวณหาค่าต่ำสุดของรายการ (list) กัน

## Getting Started (เริ่มต้นกันเลย)

มาเริ่มพิจารณา ชนิดข้อมูล (type) ของฟังก์ชันกันก่อน Rust บังคับให้เราต้องระบุชนิดข้อมูลของอาร์กิวเมนต์ทั้งหมด รวมถึงชนิดข้อมูลที่ส่งคืน (return type) ก่อนจะเริ่มเขียนเนื้อหา (body) ฟังก์ชันเสียอีก สำหรับฟังก์ชันหาค่าต่ำสุดนี้ เราอาจเผลอคิดไปว่ามันควรส่งคืนค่าเป็นตัวเลข แต่นั่นจะทำให้เกิดปัญหาตามมา ค่าต่ำสุดของรายการเปล่าคืออะไรล่ะ? ในเมื่อชนิดข้อมูลของฟังก์ชันระบุว่าต้องส่งคืน **อะไรสักอย่าง** เราอาจเลือกส่งค่า 0 กลับไปดื้อๆ เลยก็ได้ แต่นั่นก็ดูไม่มีเหตุผลรองรับ

สิ่งที่เราต้องการคือชนิดข้อมูลที่บอกว่าเป็น "ตัวเลข หรือไม่มีอะไรเลย" ชนิดข้อมูลลักษณะนี้ (ที่มีทางเลือกแบบแยกขาดจากกันหลายทาง) เรียกว่า "ชนิดข้อมูลแบบพีชคณิต" (Algebraic Datatype) ซึ่ง Rust ให้เรานิยามชนิดข้อมูลเหล่านี้ได้ด้วยคีย์เวิร์ด `enum`

อธิบายเพิ่มเติมเกี่ยวกับ **Algebraic Datatypes** คำว่า "พีชคณิต" อาจจะดูซับซ้อน แต่จริงๆ แล้วมันหมายถึงการที่เราสามารถประกอบชนิดข้อมูลใหม่จากชนิดข้อมูลที่มีอยู่แล้วได้ เหมือนกับการบวกและการคูณในพีชคณิตนั่นเองครับ

*   **Sum Types (ชนิดข้อมูลแบบผลรวม)** คือ `enum` ที่เรากำลังจะเรียนนี่แหละครับ มันหมายถึง "ค่านี้อาจจะเป็นอย่าง A หรือ อย่าง B หรือ อย่าง C..." เช่น `NumberOrNothing` ของเราคือ "เป็น Number(i32) หรือ เป็น Nothing"
*   **Product Types (ชนิดข้อมูลแบบผลคูณ)** คือ `struct` ที่เราจะเรียนในบทถัดๆ ไป มันหมายถึง "ค่านี้มีทั้งอย่าง A และ อย่าง B และ อย่าง C..."

หากคุณมาจากสาย C(++) ให้มองว่าชนิดข้อมูล `enum` นี้คือ `union` ที่มาพร้อมกับฟิลด์สำหรับเก็บสถานะว่าปัจจุบันกำลังใช้รูปแบบ (variant) ไหนของ union อยู่นั่นเอง ทำให้ `enum` ใน Rust ปลอดภัยกว่า `union` ใน C/C++ มาก เพราะ Rust จะบังคับให้เราตรวจสอบว่าตอนนี้ enum เก็บค่าอะไรอยู่ก่อนที่จะนำไปใช้งาน

`enum` สำหรับ "ตัวเลข หรือไม่มีอะไรเลย" อาจมีหน้าตาดังนี้

```rust
enum NumberOrNothing {
     Number(i32), // variant ที่เก็บค่าจำนวนเต็ม 32-บิต
     Nothing      // variant ที่ไม่มีค่าอะไรเลย
}
```

สังเกตว่า `i32` คือชนิดข้อมูลของจำนวนเต็ม (แบบมีเครื่องหมาย, 32-บิต) ในการเขียนชนิดข้อมูลของฟังก์ชันหาค่าต่ำสุด เราต้องการส่วนผสมอีกแค่อย่างเดียว `Vec<i32>` คือชนิดข้อมูลของอาร์เรย์ (ที่ขยายขนาดได้) ของตัวเลข และเราจะใช้มันเป็นชนิดข้อมูลรายการของเรา

สังเกตว่าใน Rust ชนิดข้อมูลที่ส่งคืนจะระบุไว้ **หลัง** อาร์กิวเมนต์ (คล้ายกับ TypeScript หรือภาษาอื่นๆ บางภาษา)

```rust
fn vec_min(vec: Vec<i32>) -> NumberOrNothing {
```

ภายในฟังก์ชัน ก่อนอื่นเราต้องมีตัวแปรสักตัวเพื่อเก็บค่าต่ำสุดที่คำนวณได้ล่าสุด เนื่องจากตอนเริ่มเรายังไม่ได้คำนวณอะไรเลย ค่านี้จึงเป็น "ตัวเลข หรือไม่มีอะไรเลย" เช่นกัน:

```rust
    let mut min = NumberOrNothing::Nothing;
```

เราไม่จำเป็นต้องกำกับชนิดข้อมูลไว้ข้างๆ `min` เพราะ Rust สามารถอนุมานเองได้อัตโนมัติ (คล้ายๆ กับ `auto` ใน C++11 หรือ `var` ใน C#) นอกจากนี้ให้สังเกต **mut** ใน Rust ตัวแปรจะเปลี่ยนแปลงค่าไม่ได้ (immutable) เป็นค่าเริ่มต้น คุณจึงต้องบอก Rust หากต้องการเปลี่ยนค่าตัวแปรในภายหลัง โดยการใส่คีย์เวิร์ด `mut` ไว้หน้าชื่อตัวแปร

ทีนี้เราต้องการ **วนซ้ำ (iterate)** ไปบนรายการ ซึ่ง Rust มีไวยากรณ์สำหรับตัววนซ้ำ (iterators) ที่ดูดีทีเดียว

```rust
    for el in vec { // `el` จะเป็นสำเนาของแต่ละสมาชิกใน `vec`
```

ในที่นี้ `el` คือสมาชิกแต่ละตัวในรายการ เราจำเป็นต้องอัปเดต `min` ตามค่านั้น แต่เราจะดึงค่าตัวเลขข้างในออกมาได้อย่างไร? นี่คือสิ่งที่ "การจับคู่รูปแบบ" (Pattern Matching) ช่วยเราได้

```rust
    match min { // `match` คล้ายกับ `switch` ในภาษาอื่น แต่ทรงพลังกว่ามาก
```

ในกรณี (arm) นี้ของ `match` ค่า `min` ยังไม่มีอะไรเลย (Nothing) ดังนั้นเราจะกำหนดให้มันเป็นตัวเลข `el` ที่เราเพิ่งวนมาเจอเป็นตัวแรก

```rust
    NumberOrNothing::Nothing => {
        min = NumberOrNothing::Number(el); // กำหนดให้ min เป็น Number ที่เก็บค่า el
    },
```

ส่วนในกรณี (arm) นี้ `min` มีค่าเป็นตัวเลข `n` อยู่แล้ว เราจึงต้องคำนวณหาค่าต่ำสุดใหม่แล้วเก็บลงไป (เราจะเขียนฟังก์ชัน `min_i32` ต่อทันทีหลังจากจบส่วนนี้)

```rust
    NumberOrNothing::Number(n) => { // ดึงค่า n ออกมาจาก Number(n)
        let new_min = min_i32(n, el);
            min = NumberOrNothing::Number(new_min);
        }
    }
```

สังเกตว่า Rust จะคอยตรวจสอบให้แน่ใจว่าคุณไม่ลืมจัดการกรณีใดๆ ใน `match` เราเรียกคุณสมบัตินี้ว่า **การจับคู่รูปแบบต้องครอบคลุมทั้งหมด (exhaustive)** นี่คือเหตุผลว่าทำไม `enum` ใน Rust ถึงปลอดภัยกว่า `union` ใน C/C++ เพราะคุณไม่สามารถพลาดที่จะจัดการกรณีใดกรณีหนึ่งไปได้เลย

สุดท้าย เราก็ส่งคืนผลลัพธ์การคำนวณกลับไป

```rust
    return min; // หรือเขียนแค่ `min` เฉยๆ ก็ได้ถ้าเป็นนิพจน์สุดท้ายของฟังก์ชัน
}
```

**Tip** ใน Rust บรรทัดสุดท้ายของฟังก์ชันที่ไม่ลงท้ายด้วย `;` จะถูกถือว่าเป็นค่าที่ส่งคืน (return value) ครับ ดังนั้น `return min;` สามารถเขียนเป็น `min` เฉยๆ ได้ในกรณีนี้ (แต่ถ้ามี `;` จะเป็น statement ไม่ใช่ expression)

ตอนนี้เมื่อเราย่อยปัญหาจนเหลือแค่การหาค่าต่ำสุดระหว่างจำนวนเต็มสองจำนวนแล้ว ก็มาเขียนฟังก์ชันนั้นกัน

```rust
fn min_i32(a: i32, b: i32) -> i32 {
    if a < b {
        return a;
    } else {
        return b;
    }
    // หรือเขียนแบบ Rust idiomatic กว่า:
    // if a < b { a } else { b }
}
```

เฮ้อ เขียนฟังก์ชัน Rust แรกเสร็จซะที แต่พวก `NumberOrNothing::` ที่ต้องเขียนเต็มยศนี่เริ่มดูรกๆ แล้วสิ เราจะเขียนให้ดูดีกว่านี้ได้ไหม?

จริงๆ แล้วทำได้ครับ บรรทัดต่อไปนี้บอกให้ Rust นำคอนสตรัคเตอร์ของ `NumberOrNothing` เข้ามาในเนมสเปซ (namespace) ของเรา ลองย้ายบรรทัดนี้ไปไว้เหนือฟังก์ชัน และลบ `NumberOrNothing::` ที่ปรากฏอยู่ออกทั้งหมดดูสิ

```rust
use self::NumberOrNothing::{Number,Nothing};
// หรือถ้าต้องการนำทุก variant เข้ามา:
// use self::NumberOrNothing::*;
```

`use` เป็นคีย์เวิร์ดสำหรับนำ Path (เส้นทาง) ไปยังโมดูล, ชนิดข้อมูล, ฟังก์ชัน, หรือในกรณีนี้คือ Variant ของ enum เข้ามาในขอบเขต (scope) ปัจจุบัน เพื่อให้เราสามารถเรียกใช้ได้โดยไม่ต้องระบุ Path เต็ม

ในการเรียกใช้ฟังก์ชันนี้ เราต้องการแค่รายการ (list) จริงๆ แล้วเราอยากให้ผู้ใช้เป็นคนป้อนรายการตัวเลขเข้ามา แต่สำหรับตอนนี้ขอฝังค่า (hard-code) ลงไปตรงๆ ก่อนแล้วกัน

`vec!` คือ **มาโคร (macro)** (สังเกตได้จาก `!`) ที่ช่วยสร้าง `Vec<i32>` แบบค่าคงที่ตามสมาชิกที่เรากำหนด มาโครใน Rust คล้ายกับฟังก์ชัน แต่ทำงานในระดับการคอมไพล์ โดยจะถูกขยายโค้ดก่อนที่คอมไพเลอร์จะทำงานจริง

```rust
fn read_vec() -> Vec<i32> {
    vec![18,5,7,1,9,27]
}
```

แน่นอนว่าเราย่อมอยากเห็นผลลัพธ์การคำนวณจริงๆ ด้วย ดังนั้นเราต้องสั่งพิมพ์ผลลัพธ์ออกมา Rust พิมพ์ตัวเลขได้อยู่แล้ว แต่หลังจากเรียก `vec_min` สิ่งที่เราได้คือ `NumberOrNothing` ดังนั้นเรามาเขียนฟังก์ชันช่วยทำงาน (helper function) เล็กๆ เพื่อพิมพ์ค่าชนิดนี้กันเถอะ

`println!` ก็เป็นมาโครเช่นกัน โดยอาร์กิวเมนต์ตัวแรกคือ **สตริงระบุรูปแบบ (format string)** สำหรับตอนนี้ รู้แค่ว่า `{}` คือตัวยึดตำแหน่ง (placeholder) สำหรับวางค่าลงไป และ Rust จะตรวจสอบตั้งแต่ช่วงเวลาคอมไพล์ (compile-time) ว่าคุณใส่อาร์กิวเมนต์มาครบจำนวนหรือไม่

```rust
fn print_number_or_nothing(n: NumberOrNothing) {
    match n {
        Nothing => println!("The number is: <nothing>"),
        Number(n) => println!("The number is: {}", n), // ใช้ {} เพื่อพิมพ์ค่าของ n
    };
}
```

เมื่อนำทุกอย่างมารวมกันจะได้แบบนี้

```rust
enum NumberOrNothing {
     Number(i32),
     Nothing
}
use self::NumberOrNothing::{Number,Nothing}; // นำ variant เข้ามาใน scope
fn vec_min(vec: Vec<i32>) -> NumberOrNothing {
    let mut min = Nothing; // เขียนสั้นลงได้แล้ว
    for el in vec {
        match min {
            Nothing => { // เขียนสั้นลง
                min = Number(el); // เขียนสั้นลง
            },
            Number(n) => { // เขียนสั้นลง
                let new_min = min_i32(n, el);
                min = Number(new_min); // เขียนสั้นลง
            }
        }
    }
    min // ไม่ต้องมี return; ก็ได้
}
fn min_i32(a: i32, b: i32) -> i32 {
    if a < b {
        a // ไม่ต้องมี return; ก็ได้
    } else {
        b // ไม่ต้องมี return; ก็ได้
    }
}
fn read_vec() -> Vec<i32> {
    vec![18,5,7,1,9,27]
}
fn print_number_or_nothing(n: NumberOrNothing) {
    match n {
        Nothing => println!("The number is: <nothing>"),
        Number(n) => println!("The number is: {}", n),
    };
}
pub fn main() {
    let vec = read_vec();
    let min = vec_min(vec);
    print_number_or_nothing(min);
}
```

วิธีการรันโค้ด

1.  สร้างโปรเจกต์ Rust ใหม่
    ```bash
    cargo new my_rust_project
    cd my_rust_project
    ```
2.  เปิดไฟล์ `src/main.rs` และคัดลอกโค้ดทั้งหมดข้างต้นลงไป
3.  คอมไพล์เครต (crate) ของคุณ
    ```bash
    cargo build
    ```
    (Crate คือชื่อที่ Rust ใช้เรียกหน่วยการคอมไพล์ (compilation unit) ซึ่งหมายถึงแอปพลิเคชันหรือไลบรารีนั่นเอง)
4.  สุดท้าย ลองสั่ง `cargo run` บนคอนโซลเพื่อรันโปรแกรมดู
    ```bash
    cargo run
    ```
