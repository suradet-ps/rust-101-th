# Rust-101, ส่วนที่ 00: ชนิดข้อมูลพีชคณิต (Algebraic datatypes)

สำหรับโค้ด Rust ชิ้นแรกของเรา เราต้องการเขียนฟังก์ชันที่คำนวณค่าต่ำสุดของรายการ (list)

---

## เริ่มต้นกันเลย (Getting started)

ให้เราเริ่มจากการคิดถึงชนิดข้อมูล (Type) ของฟังก์ชันเราก่อน Rust บังคับให้เราต้องระบุชนิดข้อมูลของอาร์กิวเมนต์ทั้งหมด และชนิดข้อมูลที่ส่งกลับ ก่อนที่เราจะเริ่มเขียนเนื้อหาของฟังก์ชันเสียอีก ในกรณีของฟังก์ชันหาค่าต่ำสุดของเรา เราอาจจะเอนเอียงไปบอกว่ามันคืนค่าเป็นตัวเลข แต่แบบนั้นเราจะมีปัญหา: ค่าต่ำสุดของรายการเปล่าคืออะไร? ชนิดข้อมูลของฟังก์ชันบอกว่าเราต้องส่งค่าบางอย่างกลับมา เราอาจจะเลือกส่ง 0 กลับมา แต่นั่นดูจะเป็นการเลือกแบบส่งเดชไปหน่อย สิ่งที่เราต้องการคือชนิดข้อมูลที่เป็น "ตัวเลข หรือ ความว่างเปล่า (Number or Nothing)" ชนิดข้อมูลแบบนี้ (ที่มีทางเลือกแยกจากกันหลายทาง) เรียกว่า "ชนิดข้อมูลพีชคณิต (Algebraic datatype)" และ Rust อนุญาตให้เรากำหนดชนิดข้อมูลแบบนี้ได้ด้วยคีย์เวิร์ด `enum` หากคุณมาจากภาษา C(++) คุณสามารถมองว่าชนิดข้อมูลแบบนี้เป็น `union` ที่มาพร้อมกับฟิลด์ที่เก็บว่าตอนนี้กำลังใช้งานรูปแบบ (variant) ไหนของ union อยู่

`enum` สำหรับ "ตัวเลข หรือ ความว่างเปล่า" อาจหน้าตาประมาณนี้:

```rust
enum NumberOrNothing {
    Number(i32),
    Nothing
}
```

สังเกตว่า `i32` คือชนิดข้อมูลของจำนวนเต็ม (แบบมีเครื่องหมาย, ขนาด 32 บิต) ในการเขียนชนิดข้อมูลของฟังก์ชันหาค่าต่ำสุด เราต้องการส่วนประกอบอีกเพียงอย่างเดียว: `Vec<i32>` คือชนิดข้อมูลของอาเรย์ตัวเลข (ที่ยืดขยายขนาดได้) และเราจะใช้มันเป็นชนิดข้อมูลรายการ (list) ของเรา

สังเกตดูว่าใน Rust ชนิดข้อมูลที่ส่งกลับจะอยู่ *ข้างหลัง* อาร์กิวเมนต์

```rust
fn vec_min(vec: Vec<i32>) -> NumberOrNothing {
    // ในฟังก์ชัน ขั้นแรกเราต้องมีตัวแปรสักตัวเพื่อเก็บค่าต่ำสุดที่คำนวณได้จนถึงตอนนี้
    // เนื่องจากเราเริ่มต้นโดยที่ยังไม่ได้คำนวณอะไรเลย ค่านี้ก็จะเป็น "ตัวเลข หรือ ความว่างเปล่า" เช่นกัน:
    let mut min = NumberOrNothing::Nothing;
    // เราไม่จำเป็นต้องเขียนชนิดข้อมูลกำกับไว้ข้างๆ `min` Rust สามารถรู้ได้เองโดยอัตโนมัติ
    // (คล้ายๆ กับ `auto` ใน C++11) สังเกต `mut` ด้วย: ใน Rust ตัวแปรจะมีความคงตัว (immutable)
    // เป็นค่าตั้งต้น และคุณต้องบอก Rust ถ้าหากคุณต้องการเปลี่ยนแปลงตัวแปรในภายหลัง
    
    // ตอนนี้เราต้องการ *วนซ้ำ (iterate)* ไปบนรายการ Rust มีไวยากรณ์ที่ดีสำหรับตัววนซ้ำ (iterators):
    for el in vec {
        // ดังนั้น `el` คือสมาชิกตัวหนึ่งในรายการ เราจำเป็นต้องอัปเดต `min` ตามความเหมาะสม แต่เราจะทำอย่างไร
        // ถึงจะเอาตัวเลขปัจจุบันออกมาจากในนั้นได้? นี่คือสิ่งที่การจับคู่รูปแบบ (pattern matching) ทำได้:
        match min {
            // ในกรณีนี้ (*แขน* หรือ *arm*) ของ `match` ค่า `min` ปัจจุบันคือความว่างเปล่า ดังนั้นเราก็แค่ทำให้
            // มันเป็นตัวเลข `el`
            NumberOrNothing::Nothing => {
                min = NumberOrNothing::Number(el);
            },
            // ในแขนนี้ `min` ปัจจุบันคือตัวเลข `n` ดังนั้นเรามาคำนวณค่าต่ำสุดใหม่และ
            // เก็บมันไว้ เราจะเขียนฟังก์ชัน `min_i32` ทันทีหลังจากเราเขียนฟังก์ชันนี้เสร็จ
            NumberOrNothing::Number(n) => {
                let new_min = min_i32(n, el);
                min = NumberOrNothing::Number(new_min);
            }
        }
    }
    // สังเกตว่า Rust จะคอยมั่นใจว่าคุณไม่ได้ลืมจัดการกรณีใดกรณีหนึ่งใน `match` ของคุณ
    // เราเรียกสิ่งนี้ว่า การจับคู่รูปแบบต้องครอบคลุมทุกกรณี (exhaustiveness)
    
    // สุดท้าย เราส่งผลลัพธ์ของการคำนวณกลับไป
    return min;
}
```

ตอนนี้เราได้ลดรูปปัญหาลงเหลือแค่การคำนวณค่าต่ำสุดของจำนวนเต็มสองจำนวนแล้ว เรามาทำส่วนนั้นกัน

```rust
fn min_i32(a: i32, b: i32) -> i32 {
    if a < b {
        return a;
    } else {
        return b;
    }
}
```

เฮ้อ เราเขียนฟังก์ชัน Rust ฟังก์ชันแรกของเราเสร็จแล้ว! แต่ไอ้เจ้า `NumberOrNothing::` นี่เริ่มจะดูรกหูรกตาแล้วสิ เราทำให้มันดูดีกว่านี้ไม่ได้เหรอ?

ทำได้แน่นอน: บรรทัดต่อไปนี้บอกให้ Rust นำตัวสร้าง (constructors) ของ `NumberOrNothing` เข้ามาในเนมสเปซ (namespace) ปัจจุบัน ลองย้ายบรรทัดนี้ไปไว้เหนือฟังก์ชันดู และลบคำว่า `NumberOrNothing::` ที่ปรากฏอยู่ออกทั้งหมด

```rust
use self::NumberOrNothing::{Number, Nothing};
```

ในการเรียกใช้ฟังก์ชันนี้ เราต้องการแค่รายการ (list) สักอัน แน่นอนว่าในท้ายที่สุดเราคงอยากจะขอรายการตัวเลขจากผู้ใช้ แต่สำหรับตอนนี้ เรามาเขียนโค้ดฝังค่า (hard-code) ลงไปก่อนละกัน

`vec!` คือแมโคร (สังเกตได้จาก `!`) ที่สร้าง `Vec<>` ค่าคงที่ขึ้นมาจากสมาชิกที่กำหนดให้

```rust
fn read_vec() -> Vec<i32> {
    vec![18, 5, 7, 1, 9, 27]
}
```

แน่นอนว่า เราคงอยากจะเห็นผลลัพธ์ของการคำนวณจริงๆ ดังนั้นเราต้องพิมพ์ผลลัพธ์ออกมา แน่นอนว่า Rust สามารถพิมพ์ตัวเลขได้ แต่หลังจากเรียก `vec_min` เราได้ค่าแบบ `NumberOrNothing` ดังนั้นเรามาเขียนฟังก์ชันผู้ช่วยเล็กๆ เพื่อพิมพ์ค่าเหล่านี้กันดีกว่า

`println!` ก็เป็นแมโครอีกเช่นกัน โดยอาร์กิวเมนต์ตัวแรกคือสตริงจัดรูปแบบ (format string) สำหรับตอนนี้ คุณแค่ต้องรู้ว่า `{}` คือตัวยึดตำแหน่ง (placeholder) สำหรับค่า และ Rust จะตรวจสอบตอนคอมไพล์ (compile-time) ว่าคุณใส่อาร์กิวเมนต์มาครบตามจำนวนที่ถูกต้องหรือไม่

```rust
fn print_number_or_nothing(n: NumberOrNothing) {
    match n {
        Nothing => println!("ตัวเลขคือ: <ความว่างเปล่า>"),
        Number(n) => println!("ตัวเลขคือ: {}", n),
    };
}
```

นำทุกอย่างมารวมกัน:

```rust
pub fn main() {
    let vec = read_vec();
    let min = vec_min(vec);
    print_number_or_nothing(min);
}
```

ตอนนี้คุณสามารถใช้ `cargo build` เพื่อคอมไพล์เครต (crate) ของคุณได้แล้ว นั่นคือชื่อที่ Rust เรียกหน่วยการคอมไพล์ (compilation unit) ซึ่งในกรณีของ Rust มันหมายถึงแอปพลิเคชันหรือไลบรารี สุดท้าย ลองสั่ง `cargo run` บนคอนโซลเพื่อรันมันดู

เย้ มันบอกว่า "1"! นั่นคือคำตอบที่ถูกต้อง โอเค เราอาจจะคำนวณเองก็ได้แต่นั่นไม่ใช่ประเด็น ที่สำคัญกว่านั้นคือ: คุณเรียนส่วนแรกของคอร์สจบแล้ว!
