# บทที่ 00: ชนิดข้อมูลพีชคณิต (Algebraic datatypes)

สำหรับบทเรียนแรกนี้ เราจะมาเขียนโค้ด Rust ชิ้นแรกกันครับ ซึ่งสิ่งที่เราจะเขียนก็คือคือฟังก์ชันหาค่าต่ำสุดของรายการ (list) นั่นเอง

## มาเริ่มต้นกันเลยครับ

จำไว้เสมอครับว่าก่อนที่เราจะเริ่มเขียนฟังก์ชันใดๆ ใน ภาษา Rust จะบังคับให้เราคิดเรื่องชนิดข้อมูล (data types) ของฟังก์ชันกันก่อนนั้นๆ ก่อนเ Rust บังคับให้ระบุชนิดข้อมูลของอาร์กิวเมนต์ทั้งหมดและค่าที่ส่งกลับ ก่อนจะเริ่มเขียนเนื้อฟังก์ชันเสียอีก ในกรณีของฟังก์ชันหาค่าต่ำสุด พอคิดเร็วๆ เราอาจจะบอกว่ามันคืนค่าเป็นตัวเลข แต่แบบนั้นจะมีปัญหา ค่าต่ำสุดของรายการว่างคืออะไร? ชนิดข้อมูลของฟังก์ชันบอกว่าต้องส่งค่าบางอย่างกลับมา จะเลือกส่ง 0 กลับมาก็ได้ แต่นั่นดูจะเป็นการเลือกแบบส่งเดชไปหน่อย สิ่งที่ต้องการจริงๆ คือชนิดข้อมูลที่เป็น "มีตัวเลข หรือ ไม่มีค่าเลย" ชนิดข้อมูลแบบนี้ (ที่มีทางเลือกแยกจากกันหลายทาง) เรียกว่า "ชนิดข้อมูลพีชคณิต (algebraic datatype)" ซึ่ง Rust ให้นิยามชนิดข้อมูลแบบนี้ได้ด้วยคีย์เวิร์ด `enum` สำหรับคนที่มาจากภาษา C(++) ให้นึกภาพว่ามันเป็น `union` ที่มีฟิลด์บอกว่ากำลังใช้ variant ไหนอยู่

`enum` สำหรับ "ตัวเลข หรือ ไม่มีค่า" อาจหน้าตาประมาณนี้:

```rust
enum NumberOrNothing {
    Number(i32),
    Nothing
}
```

สังเกตว่า `i32` คือชนิดข้อมูลของจำนวนเต็มแบบมีเครื่องหมาย ขนาด 32 บิต ในการเขียนชนิดข้อมูลของฟังก์ชันหาค่าต่ำสุด ยังต้องการส่วนประกอบอีกอย่าง: `Vec<i32>` คือชนิดข้อมูลของอาเรย์ตัวเลขที่ขยายขนาดได้ ซึ่งเราจะใช้เป็นชนิดข้อมูลรายการ (list) ของเรา

สังเกตดูว่าใน Rust ชนิดข้อมูลที่ส่งกลับจะอยู่ *หลัง* อาร์กิวเมนต์

```rust
fn vec_min(vec: Vec<i32>) -> NumberOrNothing {
    // ขั้นแรก ต้องมีตัวแปรสักตัวเพื่อเก็บค่าต่ำสุดที่คำนวณได้จนถึงตอนนี้
    // เนื่องจากยังไม่ได้คำนวณอะไรเลย ค่าเริ่มต้นก็จะเป็น "ตัวเลข หรือ ไม่มีค่า" เช่นกัน:
    let mut min = NumberOrNothing::Nothing;
    // ไม่จำเป็นต้องเขียนชนิดข้อมูลกำกับไว้ข้างๆ `min` เพราะ Rust อนุมานได้เอง
    // (คล้ายๆ กับ `auto` ใน C++11) สังเกต `mut` ด้วย ใน Rust ตัวแปรจะเปลี่ยนค่าไม่ได้
    // (immutable) โดยปริยาย ต้องระบุ `mut` ไว้ถ้าต้องการเปลี่ยนค่าในภายหลัง
    
    // ต่อมาจะ *วนลูป (iterate)* ผ่านทุกสมาชิกในรายการ ซึ่ง Rust มี syntax ที่กระชับดี:
    for el in vec {
        // ตอนนี้ `el` คือสมาชิกตัวหนึ่งในรายการ ต้องอัปเดต `min` ให้สอดคล้อง แต่จะดึง
        // ตัวเลขปัจจุบันออกมาจาก `min` ได้อย่างไร? ตรงนี้แหละที่ pattern matching จะมาช่วย:
        match min {
            // ในกรณี (arm) นี้ของ `match` ค่า `min` ปัจจุบันคือไม่มีค่า ก็แค่กำหนดให้
            // เป็นตัวเลข `el`
            NumberOrNothing::Nothing => {
                min = NumberOrNothing::Number(el);
            },
            // ในกรณีนี้ `min` ปัจจุบันคือตัวเลข `n` ก็คำนวณค่าต่ำสุดใหม่แล้ว
            // เก็บไว้ ส่วนฟังก์ชัน `min_i32` จะเขียนทันทีหลังจากฟังก์ชันนี้เสร็จ
            NumberOrNothing::Number(n) => {
                let new_min = min_i32(n, el);
                min = NumberOrNothing::Number(new_min);
            }
        }
    }
    // สังเกตว่า Rust จะตรวจให้แน่ใจว่าไม่มีกรณีไหนใน `match` ที่ถูกลืม
    // เรียกว่า pattern matching ต้องครอบคลุมทุกกรณี (exhaustive)
    
    // สุดท้าย ส่งผลลัพธ์กลับไป
    return min;
}
```

ตอนนี้ลดรูปปัญหาลงเหลือแค่การหาค่าต่ำสุดของจำนวนเต็มสองจำนวนแล้ว มาเขียนส่วนนั้นกัน

```rust
fn min_i32(a: i32, b: i32) -> i32 {
    if a < b {
        return a;
    } else {
        return b;
    }
}
```

เฮ้อ เขียนฟังก์ชัน Rust แรกเสร็จแล้ว แต่ไอ้เจ้า `NumberOrNothing::` นี่เริ่มจะดูรกหูรกตาแล้วสิ ทำให้มันดูดีกว่านี้ไม่ได้เหรอ?

ได้แน่นอน บรรทัดต่อไปนี้บอกให้ Rust นำ constructor ของ `NumberOrNothing` เข้ามาใน namespace ปัจจุบัน ลองย้ายบรรทัดนี้ไปไว้เหนือฟังก์ชัน แล้วลบ `NumberOrNothing::` ที่ปรากฏอยู่ออกทั้งหมดดู

```rust
use self::NumberOrNothing::{Number, Nothing};
```

ในการเรียกใช้ฟังก์ชันนี้ ต้องมีรายการ (list) สักอัน แน่นอนว่าในท้ายที่สุดคงอยากรับรายการตัวเลขจากผู้ใช้ แต่ตอนนี้เขียนฝังค่า (hard-code) ลงไปก่อนก็แล้วกัน

`vec!` คือแมโคร (สังเกตได้จาก `!`) ที่สร้าง `Vec<>` จากค่าที่กำหนดไว้ล่วงหน้า

```rust
fn read_vec() -> Vec<i32> {
    vec![18, 5, 7, 1, 9, 27]
}
```

เราคงอยากเห็นผลลัพธ์ของการคำนวณด้วย จึงต้องพิมพ์ผลลัพธ์ออกมา Rust พิมพ์ตัวเลขได้อยู่แล้ว แต่หลังจากเรียก `vec_min` สิ่งที่ได้กลับมาคือ `NumberOrNothing` ดังนั้นมาเขียนฟังก์ชันเล็กๆ ช่วยพิมพ์ค่าพวกนี้กัน

`println!` ก็เป็นแมโครเช่นกัน โดยอาร์กิวเมนต์ตัวแรกคือ format string สำหรับตอนนี้แค่รู้ว่า `{}` คือ placeholder สำหรับค่า และ Rust จะตรวจสอบตอนคอมไพล์ว่าใส่อาร์กิวเมนต์มาครบตามจำนวนหรือไม่

```rust
fn print_number_or_nothing(n: NumberOrNothing) {
    match n {
        Nothing => println!("ตัวเลขคือ: <ไม่มีค่า>"),
        Number(n) => println!("ตัวเลขคือ: {}", n),
    };
}
```

นำทุกอย่างมารวมกัน:

```rust
pub fn main() {
    let vec = read_vec();
    let min = vec_min(vec);
    print_number_or_nothing(min);
}
```

ตอนนี้ใช้ `cargo build` เพื่อคอมไพล์เครต (crate) ได้แล้ว ซึ่ง crate คือชื่อที่ Rust เรียกหน่วยการคอมไพล์ (compilation unit) หมายถึงแอปพลิเคชันหรือไลบรารี สุดท้าย ลองสั่ง `cargo run` บนคอนโซลเพื่อรันดู
