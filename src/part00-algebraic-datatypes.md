## Part 00: Algebraic datatypes
สำหรับโค้ด Rust ชิ้นแรกของเรา เราจะมาเขียนฟังก์ชันคำนวณหาค่าต่ำสุดของรายการ (list) กัน

### Getting started
มาเริ่มพิจารณา *ชนิดข้อมูล (type)* ของฟังก์ชันกันก่อน Rust บังคับให้เราต้องระบุชนิดข้อมูลของอาร์กิวเมนต์ทั้งหมด รวมถึงชนิดข้อมูลที่ส่งคืน (return type) ก่อนจะเริ่มเขียนเนื้อหา (body) ฟังก์ชันเสียอีก สำหรับฟังก์ชันหาค่าต่ำสุดนี้ เราอาจเผลอคิดไปว่ามันควรส่งคืนค่าเป็นตัวเลข แต่นั่นจะทำให้เกิดปัญหาตามมา: ค่าต่ำสุดของรายการเปล่าคืออะไรล่ะ? ในเมื่อชนิดข้อมูลของฟังก์ชันระบุว่าต้องส่งคืน *อะไรสักอย่าง* เราอาจเลือกส่งค่า 0 กลับไปดื้อๆ เลยก็ได้ แต่นั่นก็ดูไม่มีเหตุผลรองรับ (arbitrary) สิ่งที่เราต้องการคือชนิดข้อมูลที่บอกว่าเป็น "ตัวเลข หรือไม่มีอะไรเลย" ชนิดข้อมูลลักษณะนี้ (ที่มีทางเลือกแบบแยกขาดจากกันหลายทาง) เรียกว่า "ชนิดข้อมูลแบบพีชคณิต" (algebraic datatype) ซึ่ง Rust ให้เรานิยามชนิดข้อมูลเหล่านี้ได้ด้วยคีย์เวิร์ด `enum` หากคุณมาจากสาย C(++) ให้มองว่าชนิดข้อมูลนี้คือ `union` ที่มาพร้อมกับฟิลด์สำหรับเก็บสถานะว่าปัจจุบันกำลังใช้รูปแบบ (variant) ไหนของ union อยู่นั่นเอง

`enum` สำหรับ "ตัวเลข หรือไม่มีอะไรเลย" อาจมีหน้าตาดังนี้:

```rust
enum NumberOrNothing {
     Number(i32),
     Nothing
}
```

สังเกตว่า `i32` คือชนิดข้อมูลของจำนวนเต็ม (แบบมีเครื่องหมาย, 32-บิต) ในการเขียนชนิดข้อมูลของฟังก์ชันหาค่าต่ำสุด เราต้องการส่วนผสมอีกแค่อย่างเดียว: `Vec<i32>` คือชนิดข้อมูลของอาร์เรย์ (ที่ขยายขนาดได้) ของตัวเลข และเราจะใช้มันเป็นชนิดข้อมูลรายการของเรา

สังเกตว่าใน Rust ชนิดข้อมูลที่ส่งคืนจะระบุไว้ *หลัง* อาร์กิวเมนต์

```rust
fn vec_min(vec: Vec<i32>) -> NumberOrNothing {
```

ภายในฟังก์ชัน ก่อนอื่นเราต้องมีตัวแปรสักตัวเพื่อเก็บค่าต่ำสุดที่คำนวณได้ล่าสุด เนื่องจากตอนเริ่มเรายังไม่ได้คำนวณอะไรเลย ค่านี้จึงเป็น "ตัวเลข หรือไม่มีอะไรเลย" เช่นกัน:

```rust
    let mut min = NumberOrNothing::Nothing;
```

เราไม่จำเป็นต้องกำกับชนิดข้อมูลไว้ข้างๆ `min` เพราะ Rust สามารถอนุมานเองได้อัตโนมัติ (คล้ายๆ กับ `auto` ใน C++11) นอกจากนี้ให้สังเกต `mut`: ใน Rust ตัวแปรจะเปลี่ยนแปลงค่าไม่ได้ (immutable) เป็นค่าเริ่มต้น คุณจึงต้องบอก Rust หากต้องการเปลี่ยนค่าตัวแปรในภายหลัง

ทีนี้เราต้องการ *วนซ้ำ (iterate)* ไปบนรายการ ซึ่ง Rust มีไวยากรณ์สำหรับตัววนซ้ำ (iterators) ที่ดูดีทีเดียว:

```rust
    for el in vec {
```

ในที่นี้ `el` คือสมาชิกแต่ละตัวในรายการ เราจำเป็นต้องอัปเดต `min` ตามค่านั้น แต่เราจะดึงค่าตัวเลขข้างในออกมาได้อย่างไร? นี่คือสิ่งที่การจับคู่รูปแบบ (pattern matching) ช่วยเราได้:

```rust
    match min {
```

ในกรณี (arm) นี้ของ `match` ค่า `min` ยังไม่มีอะไรเลย ดังนั้นเราจะกำหนดให้มันเป็นตัวเลข `el`

```rust
    NumberOrNothing::Nothing => {
        min = NumberOrNothing::Number(el);
    },
```

ส่วนในกรณี (arm) นี้ `min` มีค่าเป็นตัวเลข `n` อยู่แล้ว เราจึงต้องคำนวณหาค่าต่ำสุดใหม่แล้วเก็บลงไป (เราจะเขียนฟังก์ชัน `min_i32` ต่อทันทีหลังจากจบส่วนนี้)

```rust
    NumberOrNothing::Number(n) => {
        let new_min = min_i32(n, el);
            min = NumberOrNothing::Number(new_min);
        }
    }
```

สังเกตว่า Rust จะคอยตรวจสอบให้แน่ใจว่าคุณไม่ลืมจัดการกรณีใดๆ ใน `match` เราเรียกคุณสมบัตินี้ว่า การจับคู่รูปแบบต้องครอบคลุมทั้งหมด (exhaustive)

สุดท้าย เราก็ส่งคืนผลลัพธ์การคำนวณกลับไป

```rust
    return min;
}
```

ตอนนี้เมื่อเราย่อยปัญหาจนเหลือแค่การหาค่าต่ำสุดระหว่างจำนวนเต็มสองจำนวนแล้ว ก็มาเขียนฟังก์ชันนั้นกัน

```rust
fn min_i32(a: i32, b: i32) -> i32 {
    if a < b {
        return a;
    } else {
        return b;
    }
}
```

เฮ้อ (Phew) เขียนฟังก์ชัน Rust แรกเสร็จซะที! แต่พวก `NumberOrNothing::` ที่ต้องเขียนเต็มยศนี่เริ่มดูรกๆ แล้วสิ (ugly) เราจะเขียนให้ดูดีกว่านี้ (nicer) ได้ไหม?

จริงๆ แล้วทำได้ครับ: บรรทัดต่อไปนี้บอกให้ Rust นำคอนสตรัคเตอร์ของ `NumberOrNothing` เข้ามาในเนมสเปซ (namespace) ของเรา ลองย้ายบรรทัดนี้ไปไว้เหนือฟังก์ชัน และลบ `NumberOrNothing::` ที่ปรากฏอยู่ออกทั้งหมดดูสิ

```rust
use self::NumberOrNothing::{Number,Nothing};
```

ในการเรียกใช้ฟังก์ชันนี้ เราต้องการแค่รายการ (list) จริงๆ แล้วเราอยากให้ผู้ใช้เป็นคนป้อนรายการตัวเลขเข้ามา แต่สำหรับตอนนี้ขอฝังค่า (hard-code) ลงไปตรงๆ ก่อนแล้วกัน

`vec!` คือ *มาโคร* (สังเกตได้จาก `!`) ที่ช่วยสร้าง `Vec<_>` แบบค่าคงที่ตามสมาชิกที่เรากำหนด

```rust
fn read_vec() -> Vec<i32> {
    vec![18,5,7,1,9,27]
}
```

แน่นอนว่าเราย่อมอยากเห็นผลลัพธ์การคำนวณจริงๆ ด้วย ดังนั้นเราต้องสั่งพิมพ์ผลลัพธ์ออกมา Rust พิมพ์ตัวเลขได้อยู่แล้ว แต่หลังจากเรียก `vec_min` สิ่งที่เราได้คือ `NumberOrNothing` ดังนั้นเรามาเขียนฟังก์ชันช่วยทำงาน (helper function) เล็กๆ เพื่อพิมพ์ค่าชนิดนี้กันเถอะ

`println!` ก็เป็นมาโครเช่นกัน โดยอาร์กิวเมนต์ตัวแรกคือ *สตริงระบุรูปแบบ (format string)* สำหรับตอนนี้ รู้แค่ว่า `{}` คือตัวยึดตำแหน่ง (placeholder) สำหรับวางค่าลงไป และ Rust จะตรวจสอบตั้งแต่ช่วงเวลาคอมไพล์ (compile-time) ว่าคุณใส่อาร์กิวเมนต์มาครบจำนวนหรือไม่

```rust
fn print_number_or_nothing(n: NumberOrNothing) {
    match n {
        Nothing => println!("The number is: <nothing>"),
        Number(n) => println!("The number is: {}", n),
    };
}
```

เมื่อนำทุกอย่างมารวมกันจะได้แบบนี้:

```rust
enum NumberOrNothing {
     Number(i32),
     Nothing
}

use self::NumberOrNothing::{Number,Nothing};

fn vec_min(vec: Vec<i32>) -> NumberOrNothing {
    let mut min = NumberOrNothing::Nothing;

    for el in vec {
        match min {
            NumberOrNothing::Nothing => {
                min = NumberOrNothing::Number(el);
            },
            NumberOrNothing::Number(n) => {
                let new_min = min_i32(n, el);
                min = NumberOrNothing::Number(new_min);
            }
        }
    }
    return min;
}

fn min_i32(a: i32, b: i32) -> i32 {
    if a < b {
        return a;
    } else {
        return b;
    }
}

fn read_vec() -> Vec<i32> {
    vec![18,5,7,1,9,27]
}

fn print_number_or_nothing(n: NumberOrNothing) {
    match n {
        Nothing => println!("The number is: <nothing>"),
        Number(n) => println!("The number is: {}", n),
    };
}

pub fn main() {
    let vec = read_vec();
    let min = vec_min(vec);
    print_number_or_nothing(min);
}
```

ตอนนี้คุณสามารถใช้คำสั่ง `cargo build` เพื่อคอมไพล์เครต (crate) ของคุณได้แล้ว (Crate คือชื่อที่ Rust ใช้เรียกหน่วยการคอมไพล์ (compilation unit) ซึ่งหมายถึงแอปพลิเคชันหรือไลบรารีนั่นเอง)
สุดท้าย ลองสั่ง `cargo run` บนคอนโซลเพื่อรันโปรแกรมดู
