# Rust-101, ส่วนที่ 01: นิพจน์ (Expressions), เมธอดภายในตัว (Inherent methods)

เพื่อให้ Rust คอมไพล์ไฟล์นี้ได้ ตรวจสอบให้แน่ใจว่าได้เปิดใช้งานบรรทัดที่เกี่ยวข้องใน `main.rs` ก่อนที่จะดำเนินการต่อ

ถึงแม้ว่าโค้ดของเราจากส่วนแรกจะทำงานได้ แต่เรายังสามารถเรียนรู้อะไรได้อีกเยอะโดยการทำให้มันสวยงามขึ้น นั่นเป็นเพราะ Rust เป็นภาษาแบบ "ฐานนิพจน์ (expression-based)" ซึ่งหมายความว่าพจน์ (term) ส่วนใหญ่ที่คุณเขียนลงไปไม่ใช่แค่คำสั่ง (statements) (ที่สั่งรันโค้ด) แต่เป็นนิพจน์ (expressions) (ที่ส่งค่ากลับ) สิ่งนี้ใช้ได้แม้กระทั่งกับเนื้อหาของฟังก์ชันทั้งฟังก์ชันเลย!

## การเขียนโปรแกรมบนฐานนิพจน์ (Expression-based programming)

ตัวอย่างเช่น พิจารณา `sqr`:

```rust
fn sqr(i: i32) -> i32 { i * i }
```

ระหว่างวงเล็บปีกกา เรากำลังใส่นิพจน์ที่คำนวณค่าที่จะส่งกลับ ดังนั้นเราสามารถเขียนแค่ `i * i` ซึ่งเป็นนิพจน์ที่คืนค่ากำลังสองของ `i` ได้เลย! นี่ใกล้เคียงมากกับวิธีที่นักคณิตศาสตร์เขียนฟังก์ชัน (แต่มีเรื่องชนิดข้อมูลเข้ามาเพิ่มนิดหน่อย)

ประโยคเงื่อนไข (Conditionals) ก็เป็นแค่นิพจน์เช่นกัน นี่เทียบได้กับตัวดำเนินการไตรภาค (ternary operator) `?:` จากภาษาอย่าง C

```rust
fn abs(i: i32) -> i32 { if i >= 0 { i } else { -i } }
```

และหลักการเดียวกันนี้ก็ใช้กับการจำแนกกรณี (case distinction) ด้วย `match`: ทุกๆ แขน (arm) ของ `match` จะให้นิพจน์ที่จะถูกส่งค่ากลับในกรณีนั้นๆ (เราขอนำนิยามจากส่วนที่แล้วมาซ้ำตรงนี้)

```rust
enum NumberOrNothing {
    Number(i32),
    Nothing
}
use self::NumberOrNothing::{Number,Nothing};

fn number_or_default(n: NumberOrNothing, default: i32) -> i32 {
    match n {
        Nothing => default,
        Number(n) => n,
    }
}
```

มันถึงขั้นที่ว่า บล็อก (blocks) ก็เป็นนิพจน์ด้วย ซึ่งจะประเมินค่าออกมาเป็นนิพจน์สุดท้ายที่มันบรรจุอยู่

```rust
fn compute_stuff(x: i32) -> i32 {
    let y = { let z = x*x; z + 14 };
    y*y
}
```

ตอนนี้เรามาปรับโครงสร้าง (refactor) `vec_min` กันเถอะ

```rust
fn vec_min(v: Vec<i32>) -> NumberOrNothing {
    fn min_i32(a: i32, b: i32) -> i32 {
        if a < b { a } else { b }
    }

    let mut min = Nothing;
    for e in v {
        min = Number(match min {
            Nothing => e,
            Number(n) => min_i32(n, e)
        });
    }
    min
}
```

จำฟังก์ชันผู้ช่วย `min_i32` นั่นได้ไหม? Rust อนุญาตให้เรานิยามฟังก์ชันผู้ช่วยแบบนั้นไว้ *ข้างใน* ฟังก์ชันอื่นได้ นี่เป็นแค่เรื่องของการแยกเนมสเปซ (namespacing) ฟังก์ชันชั้นในจะไม่มีสิทธิ์เข้าถึงข้อมูลของฟังก์ชันชั้นนอก ถึงกระนั้น การที่สามารถจัดกลุ่มฟังก์ชันได้อย่างสวยงามก็ช่วยเพิ่มความสามารถในการอ่านโค้ดได้ (readability) อย่างมีนัยสำคัญ

สังเกตว่าสิ่งที่เราทำทั้งหมดที่นี่คือการคำนวณค่าใหม่ให้กับ `min` และมันจะลงเอยด้วยการเป็น `Number` เสมอ แทนที่จะเป็น `Nothing` ใน Rust โครงสร้างของโค้ดสามารถสื่อความหมายนี้ได้

คีย์เวิร์ด `return` มีอยู่ใน Rust แต่มันถูกใช้น้อยมาก แทนที่จะใช้มัน เรามักจะใช้ประโยชน์จากความจริงที่ว่าเนื้อหาของฟังก์ชันทั้งก้อนคือนิพจน์ ดังนั้นเราจึงสามารถเขียนค่าที่ต้องการส่งกลับลงไปได้เลย

ตอนนี้มันสั้นลงตั้งเยอะ! ตรวจสอบให้แน่ใจว่าคุณสามารถไล่ดูโค้ดข้างบนและเข้าใจจริงๆ ว่าเกิดอะไรขึ้นในทุกขั้นตอน

## การอิมพลีเมนท์ภายในตัว (Inherent implementations)

สำหรับ `vec_min` ก็มีเท่านี้ เรามาพิจารณา `print_number_or_nothing` กันใหม่ จริงๆ แล้วฟังก์ชันนั้นควรจะอยู่ใกล้ชิดกับ Type `NumberOrNothing` มากๆ ใน C++ หรือ Java คุณคงจะสร้างมันเป็นเมธอด (method) ของ Type นั้น ใน Rust เราสามารถทำสิ่งที่คล้ายกันมากได้โดยการจัดเตรียมการอิมพลีเมนท์ภายในตัว (inherent implementation):

```rust
impl NumberOrNothing {
    fn print(self) {
        match self {
            Nothing => println!("The number is: <nothing>"),
            Number(n) => println!("The number is: {}", n),
        };
    }
}
```

แล้ว เกิดอะไรขึ้นล่ะเนี่ย? Rust แยกโค้ดออกจากข้อมูล ดังนั้นการนิยามเมธอดบน `enum` (และบน `struct` ด้วย ซึ่งเราจะได้เรียนรู้กันภายหลัง) จึงเป็นอิสระจากการนิยาม Type `self` เหมือนกับ `this` ในภาษาอื่นๆ และ Type ของมันจะเป็นที่รู้กันโดยนัยเสมอ ดังนั้นตอนนี้ `print` จึงเป็นเมธอดที่รับ `NumberOrNothing` เป็นอาร์กิวเมนต์ตัวแรก เหมือนกับ `print_number_or_nothing` เป๊ะๆ

> **ลองทำให้ `number_or_default` จากข้างบนกลายเป็นเมธอดภายในตัว (inherent method) ด้วยสิ!**

ด้วยฟังก์ชันและเมธอดที่เราปรับโครงสร้างใหม่แล้ว ตอนนี้ `main` จะหน้าตาเป็นแบบนี้:

```rust
fn read_vec() -> Vec<i32> {
    vec![18, 5, 7, 2, 9, 27]
}

pub fn main() {
    let vec = read_vec();
    let min = vec_min(vec);
    min.print();
}
```

คุณจะต้องแทนที่ `part00` ด้วย `part01` ในฟังก์ชัน `main` ใน `main.rs` เพื่อรันโค้ดนี้
