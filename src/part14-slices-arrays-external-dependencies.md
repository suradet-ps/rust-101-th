# Rust-101, ส่วนที่ 14: Slices, อาร์เรย์ (Arrays), External Dependencies

เพื่อทำให้ rgrep สมบูรณ์ มีสองชิ้นส่วนที่เรายังต้องอิมพลีเมนท์: การเรียงลำดับ (Sorting), และการรับตัวเลือกของงาน (job options) เป็นอาร์กิวเมนต์เข้าสู่โปรแกรม แทนที่จะ hard-code พวกมันไว้ เริ่มจากการเรียงลำดับกันก่อน

## Slices

อีกครั้ง เราต้องคิดก่อนเกี่ยวกับชนิดข้อมูลที่เราต้องการจะมอบให้กับฟังก์ชันเรียงลำดับของเรา เราอาจจะโน้มเอียงไปทางส่ง `Vec<T>` ให้มัน แน่นอนว่า การเรียงลำดับไม่ได้บริโภคอาร์กิวเมนต์จริงๆ ดังนั้นเราควรจะใช้ `&mut Vec<T>` แต่มันมีปัญหาอยู่ตรงนั้น: ถ้าเราต้องการอิมพลีเมนท์อัลกอริทึมการเรียงลำดับแบบแบ่งแยกและเอาชนะ (divide-and-conquer) (อย่างเช่น Quicksort) เราจะต้องแบ่ง (split) อาร์กิวเมนต์ของเรา ณ จุดใดจุดหนึ่ง และทำงานแบบเรียกซ้ำ (recursively) บนทั้งสองส่วนนั้น แต่เราไม่สามารถแบ่ง `Vec` ได้! เราอาจจะขยายลายเซ็นฟังก์ชัน (function signature) ให้รับดัชนี (indices) บางตัว ที่ระบุส่วนของเวกเตอร์ที่เราควรจะเรียงลำดับ แต่นั่นดูเทอะทะไปหน่อย Rust เสนอทางแก้ที่ดีกว่า

`[T]` เป็นชนิดข้อมูลของอาร์เรย์ (ที่ไม่มีขนาด) ที่มีสมาชิกเป็นชนิดข้อมูล `T` ทั้งหมดนี้หมายความว่ามีพื้นที่หน่วยความจำที่ต่อเนื่องกัน (contiguous region of memory) โดยมีกลุ่มของ `T` ถูกเก็บอยู่ มีกี่ตัว? เราบอกไม่ได้! นี่เป็นชนิดข้อมูลที่ไม่มีขนาด (unsized type) เช่นเดียวกับ trait objects นี่หมายความว่าเราสามารถทำงานบน pointers ไปยังชนิดข้อมูลนั้นได้เท่านั้น และ pointers เหล่านี้จะถือข้อมูลที่ขาดหายไปมาด้วย - นั่นคือ ความยาว (พวกมันจะเป็น fat pointers) reference ไปยังอาร์เรย์แบบนี้เรียกว่า **slice** อย่างที่เราจะเห็น slice สามารถถูกแบ่งได้ ฟังก์ชันของเราจึงสามารถรับ mutable slice และสัญญาว่าจะเรียงลำดับสมาชิกทั้งหมดในนั้น

```rust
pub fn sort<T: PartialOrd>(data: &mut [T]) {
    if data.len() < 2 { return; }

    // เราตัดสินใจว่าสมาชิกที่ 0 คือ pivot (จุดหมุน) ของเรา และจากนั้นเราเลื่อน cursors ของเราผ่านส่วนที่เหลือ
    // ของ slice เพื่อให้แน่ใจว่าทุกอย่างทางซ้ายไม่มีค่าใหญ่กว่า pivot และ
    // ทุกอย่างทางขวาไม่มีค่าเล็กกว่า
    let mut lpos = 1;
    let mut rpos = data.len();
    /* Invariant: pivot is data[0]; everything with index (0,lpos) is <= pivot;
       [rpos,len) is >= pivot; lpos < rpos */
    loop {
        // **แบบฝึกหัด 14.1**: เติมลูป Quicksort นี้ให้สมบูรณ์ คุณสามารถใช้ `swap` บน slices เพื่อสลับ
        // สมาชิกสองตัว เขียนฟังก์ชันทดสอบสำหรับ `sort`
        unimplemented!()
    }

    // เมื่อ cursors ของเรามาเจอกัน เราต้องเอา pivot ไปวางในที่ที่ถูกต้อง
    data.swap(0, lpos-1);

    // สุดท้าย เราแบ่ง slice ของเราเพื่อเรียงลำดับทั้งสองครึ่ง ส่วนที่ดีเกี่ยวกับ slices คือ
    // การแบ่งพวกมันนั้นราคาถูก: พวกมันเป็นแค่ pointer ไปยังที่อยู่เริ่มต้น และความยาว เราจึง
    // สามารถรับ pointers สองตัว ตัวหนึ่งที่จุดเริ่มต้นและอีกตัวตรงกลาง และตั้งค่าความยาว
    // อย่างเหมาะสมเพื่อไม่ให้พวกมันทับซ้อนกัน นี่คือสิ่งที่ `split_at_mut` ทำ เนื่องจากทั้งสอง
    // slices ไม่ทับซ้อนกัน จึงไม่มี aliasing และเราสามารถมีพวกมันทั้งคู่ในฐานะ unique, mutable
    // slices ได้
    let (part1, part2) = data.split_at_mut(lpos);
    // การปฏิบัติการดัชนี (index operation) ไม่เพียงสามารถใช้เพื่อระบุสมาชิกบางตัว แต่ยังสามารถใช้
    // เพื่อ slicing ได้ด้วย: โดยการให้ช่วงของดัชนี และรับส่วนที่เหมาะสมของ slice ที่เรา
    // เริ่มต้นด้วยออกมา ที่นี่ เราเอาสมาชิกตัวสุดท้ายออกจาก part1 ซึ่งก็คือตัว pivot นั่นเอง นี่ทำให้
    // มั่นใจว่าการเรียกซ้ำทั้งสองครั้งทำงานบน slices ที่เล็กลงอย่างเคร่งครัด (strictly smaller)
    sort(&mut part1[..lpos-1]);
    sort(part2);
}
```

> **แบบฝึกหัด 14.2**: เนื่องจาก `String` อิมพลีเมนท์ `PartialEq` คุณสามารถเปลี่ยนฟังก์ชัน `output_lines` ในส่วนก่อนหน้านี้ให้เรียกฟังก์ชัน sort ข้างบนได้แล้ว ถ้าคุณทำแบบฝึกหัด 13.1 คุณจะมีงานเพิ่มขึ้นเล็กน้อย จงแน่ใจว่าคุณเรียงลำดับตามบรรทัดที่ตรงกันเท่านั้น ไม่ใช่ตามชื่อไฟล์หรือเลขบรรทัด!

ตอนนี้ เราสามารถเรียงลำดับ เช่น เวกเตอร์ของตัวเลข

```rust
fn sort_nums(data: &mut Vec<i32>) {
    // เวกเตอร์รองรับการ slicing เช่นเดียวกับที่ slices ทำ ที่นี่ `..` บ่งบอกถึงช่วงเต็ม (full range) ซึ่ง
    // หมายความว่าเราต้องการ slice เวกเตอร์ทั้งตัว จากนั้นมันถูกส่งไปให้ฟังก์ชัน sort ซึ่ง
    // ไม่รู้ด้วยซ้ำว่ามันกำลังทำงานบนข้อมูลภายในเวกเตอร์
    sort(&mut data[..]);
}
```

## อาร์เรย์ (Arrays)

อาร์เรย์ใน Rust ถูกกำหนดโดยชนิดข้อมูล `[T; n]` โดยที่ `n` คือตัวเลขคงที่บางตัว ดังนั้น `[f64; 10]` คืออาร์เรย์ของตัวเลขทศนิยม 10 ตัว ทั้งหมดอยู่ติดกันและกันในหน่วยความจำ อาร์เรย์เป็นแบบมีขนาด (sized) และดังนั้นจึงสามารถใช้เหมือนชนิดข้อมูลอื่นๆ แต่เรายังสามารถยืมพวกมันเป็น slices ได้ เช่น เพื่อเรียงลำดับพวกมัน

```rust
fn sort_array() {
    let mut array_of_data: [f64; 5] = [1.0, 3.4, 12.7, -9.12, 0.1];
    sort(&mut array_of_data);
}
```

## External Dependencies (ไลบรารีภายนอก)

นี่ทิ้งเราไว้กับอีกเพียงชิ้นส่วนเดียวเพื่อทำให้ rgrep สมบูรณ์: การรับอาร์กิวเมนต์จาก commandline ตอนนี้เราสามารถทำงานโดยตรงบน `std::env::args` เพื่อให้เข้าถึงอาร์กิวเมนต์เหล่านั้น และนี่จะกลายเป็นบทเรียนการจัดการสตริงที่ค่อนข้างน่าเบื่อ แทนที่จะเป็นอย่างนั้น ผมต้องการใช้โอกาสนี้เพื่อแสดงให้เห็นว่ามันง่ายแค่ไหนที่จะได้รับประโยชน์จากงานของคนอื่นในโปรแกรมของคุณ

แน่นอนว่า เราไม่ใช่คนแรกที่ติดตั้งการรองรับ command-line arguments ให้กับโปรแกรม Rust ต้องมีใครสักคนเขียนไลบรารีสำหรับงานนี้ไว้แล้วใช่ไหม? จริงอย่างนั้น มีคนทำแล้ว Rust มีคลังเก็บกลาง (central repository) ของไลบรารีที่เผยแพร่แล้ว เรียกว่า [crates.io](https://crates.io) มันคล้ายๆ กับ PyPI หรือ Ruby Gems: ทุกคนสามารถอัปโหลดโค้ดของพวกเขา และมีเครื่องไม้เครื่องมือสำหรับ import โค้ดนั้นเข้ามาในโปรเจกต์ของคุณ เครื่องมือนี้ถูกจัดเตรียมโดย `cargo` ซึ่งเป็นเครื่องมือที่เราใช้อยู่แล้วเพื่อ build บทเรียนนี้ (`cargo` ยังรองรับการเผยแพร่ crate ของคุณบน crates.io ด้วย ผมขออ้างถึงเอกสารสำหรับรายละเอียดเพิ่มเติม) ในกรณีนี้ เราจะใช้ crate `docopt` ซึ่งสร้างตัวแจงส่วน (parser) สำหรับ command-line arguments โดยอิงตาม usage string (สตริงการใช้งาน) External dependencies จะถูกประกาศใน `Cargo.toml`

ผมเตรียมไฟล์นั้นไว้แล้ว แต่การประกาศ dependency ยังถูกคอมเมนต์ไว้อยู่ ดังนั้นกรุณาเปิด `Cargo.toml` ของ workspace ของคุณตอนนี้ และเปิดใช้งานสองบรรทัดที่ถูกคอมเมนต์ไว้ จากนั้นสั่ง `cargo build` ตอนนี้ Cargo จะดาวน์โหลด crate จาก crates.io, คอมไพล์มัน, และลิงก์มันเข้ากับโปรแกรมของคุณ ในอนาคต คุณสามารถสั่ง `cargo update` เพื่อให้มันดาวน์โหลดเวอร์ชันใหม่ของ crates ที่คุณพึ่งพาได้ สังเกตว่า crates.io เป็นเพียงตำแหน่งตั้งต้นสำหรับ dependencies คุณยังสามารถให้ URL ของ git repository หรือ path ในเครื่องได้ ทั้งหมดนี้ถูกอธิบายไว้ในเอกสารของ Cargo

ข้าพเจ้าปิดการใช้งานโมดูลต่อไปนี้ (โดยใช้ hack ที่ค่อนข้างแย่) เพราะมันจะคอมไพล์ได้ก็ต่อเมื่อ `docopt` ถูกลิงก์แล้ว ลบ attribute ของโมดูล `rgrep` ออกเพื่อเปิดใช้งานการคอมไพล์

```rust
#[cfg(feature = "disabled")]
pub mod rgrep {
    // ตอนนี้เมื่อ `docopt` ถูกลิงก์แล้ว เราสามารถเพิ่มมันเข้ามาใน namespace ด้วย `extern crate` ก่อน
    // และจากนั้น import ชื่อที่สั้นกว่าด้วย `use` เรายัง import ชิ้นส่วนอื่นๆ ที่เราจะต้องใช้ด้วย
    extern crate docopt;
    use self::docopt::Docopt;
    use part13::{run, Options, OutputMode};
    use std::process;

    // สตริง `USAGE` (การใช้งาน) เอกสารวิธีที่โปรแกรมจะถูกเรียก มันถูกเขียนในรูปแบบที่
    // `docopt` สามารถ parse ได้
    static USAGE: &'static str = "
Usage: rgrep [-c] [-s] <pattern> <file>...

Options:
    -c, --count  Count number of matching lines (rather than printing them).
    -s, --sort   Sort the lines before printing.
";

    // ฟังก์ชันนี้สกัด options ของ rgrep ออกมาจาก command-line arguments
    fn get_options() -> Options {
        // สิ่งนี้ parse `argv` และออกจากโปรแกรมพร้อมข้อความ error ถ้ามันล้มเหลว โค้ดนี้
        // ถูกนำมาจาก [`docopt` documentation](http://burntsushi.net/rustdoc/docopt/) <br/>
        // ฟังก์ชัน `and_then` รับ closure จาก `T` ไปยัง `Result<U,E>` และใช้มันเพื่อแปลง
        // `Result<T,E>` ไปเป็น `Result<U,E>` ด้วยวิธีนี้ เราสามารถลูกโซ่ (chain) การคำนวณที่จะเกิดขึ้น
        // ก็ต่อเมื่ออันก่อนหน้าสำเร็จเท่านั้น (และชนิดของ error ต้องเหมือนเดิม) ในกรณีที่คุณรู้
        // เกี่ยวกับ monads สไตล์การเขียนโปรแกรมนี้จะคุ้นเคยสำหรับคุณ มีฟังก์ชันที่คล้ายกัน
        // สำหรับ `Option` `unwrap_or_else` คล้ายกับ `unwrap` นิดหน่อย แต่แทนที่จะ panic ในกรณี
        // ที่เป็น `Err` มันจะเรียก closure
        let args = Docopt::new(USAGE).and_then(|d| d.parse()).unwrap_or_else(|e| e.exit());
        
        // ตอนนี้เราสามารถเอาค่าทั้งหมดออกมาได้
        let count = args.get_bool("-c");
        let sort = args.get_bool("-s");
        let pattern = args.get_str("<pattern>");
        let files = args.get_vec("<file>");
        if count && sort {
            println!("Setting both '-c' and '-s' at the same time does not make any sense.");
            process::exit(1);
        }

        // เราจำเป็นต้องทำให้สตริงเป็นแบบ owned เพื่อสร้าง instance ของ `Options` ถ้าคุณเช็ค
        // ชนิดข้อมูลทั้งหมดอย่างระมัดระวัง คุณจะสังเกตเห็นว่า `pattern` ข้างบนมีชนิดข้อมูล `&str` `str` คือ
        // ชนิดข้อมูลของสตริงที่เข้ารหัส UTF-8 นั่นคือ กลุ่มของไบต์ในหน่วยความจำ (`[u8]`) ที่
        // ถูกต้องตามมาตรฐาน UTF-8 `str` นั้นไม่มีขนาด (unsized) `&str` เก็บที่อยู่ของข้อมูลตัวอักษร
        // และความยาวของพวกมัน String literals อย่าง `"this one"` มีชนิดข้อมูล `&'static str`:
        // พวกมันชี้ตรงไปยังส่วนคงที่ (constant section) ของไบนารี ดังนั้น reference จึงใช้งานได้
        // ตลอดทั้งโปรแกรม ไบต์ที่ถูกชี้โดย `pattern` ในทางกลับกัน ถูกเป็นเจ้าของโดย
        // คนอื่น และเราเรียก `to_string` บนมันเพื่อคัดลอกข้อมูลสตริงเข้าไปในบัฟเฟอร์บน
        // ฮีปที่เราเป็นเจ้าของ
        let mode = if count {
            OutputMode::Count
        } else if sort {
            OutputMode::SortAndPrint
        } else {
            OutputMode::Print
        };
        Options {
            files: files.iter().map(|file| file.to_string()).collect(),
            pattern: pattern.to_string(),
            output_mode: mode,
        }
    }

    // สุดท้าย เราสามารถเรียกฟังก์ชัน `run` จากส่วนที่แล้วบน options ที่สกัดออกมาโดยใช้
    // `get_options` แก้ไข `main.rs` เพื่อเรียกฟังก์ชันนี้
    // ตอนนี้คุณสามารถใช้ `cargo run -- <pattern> <files>` เพื่อเรียกโปรแกรมของคุณ และดู argument
    // parser และเธรดที่เราเขียนก่อนหน้านี้ทำงานได้เลย!
    pub fn main() {
        run(get_options());
    }
}
```

> **แบบฝึกหัด 14.3**: มันจะดีไหมถ้า rgrep รองรับ regular expressions? มี crate ที่ทำหน้าที่ parsing และ matching บน regular expression อยู่แล้ว มันเรียกว่า [regex](https://crates.io/crates/regex) จงเพิ่ม crate นี้เข้าไปใน dependencies ของ workspace ของคุณ, เพิ่มตัวเลือก ("-r") เพื่อสลับ pattern ไปเป็นโหมด regular-expression, และเปลี่ยน `filter_lines` ให้เคารพตัวเลือกนี้ เอกสารของ regex มีให้อ่านจากหน้า crates.io ของมัน (คุณจะไม่สามารถใช้มาโคร `regex!` ถ้าคุณอยู่บน stable หรือ beta channel ของ Rust แต่มันก็ไม่ได้ช่วยอะไรสำหรับกรณีการใช้งานของเราอยู่แล้ว)
